#### 1.3.2 后台运行

区别：

- 前台运行：命令在命令行运行，无法执行其他命令，需要等待

- 后台运行：一些命令在前台运行的的时候连接断开了，或网络断开，前台运行的命令会结束。后台运行则没有影响

- & and（俺的）

  | 后台运行           | 说明                                                         |
  | ------------------ | ------------------------------------------------------------ |
  | 命令或脚本 &       | 后台运行。连接断开也结束                                     |
  | nohup 命令或脚本 & | 后台运行，连接断开也不结束，默认会把命令/脚本输出到nohup.out文件 |

- 输出到nohup.out文件

  ```sh
  # 1.后台运行
  nohup ping baidu.com &
  # 2.检查
  tail -f nohup.put
  ```

  

- 指定输出文件

  ```sh
  nohup ping baidu.com > /oldboy-jd.log &
  ```

  小结：

  - 命令/脚本 &
  - nohup 命令/脚本 &
  - 保存输出到指定文件

```sh
D	uninterruptible sleep (usually I0)
I	Idle kernel thread
R	running or runnable (on run queue)
```

```sh
命令使用
对top命令的结果进行取行与取列.

top命令默认是交互模式便于展示结果，但是不方便进行过滤或二次加工.
top命令变成非交互模式.
top -bn1

-n1 显示1次.
top -bn1 |grep 'zombie'|awk '{print $(NF-3)}'

案例:取出系统登录用户数量.
top -bn1 |awk 'NR==1'| awk '{print $(NF-6)}'
案例:取出僵尸进程数量.
top -bn1 |awk 'NR==2{print $(NF-1)}'

# 查看进程下的线程信息

```

## 三、服务管理

### 3.1 命令

- systemctl
- 设置服务是否开机自启动
- 开/关/重启服务
- 检查

#### 3.1.1 开启sshd服务

```sh
# 设置服务开机自启动
systemctl enable sshd

# 当前启动服务器
systemctl start sshd

#2和1
systemctl enable --now sshd
```

#### 3.1.2 关闭sshd服务

```sh
#设置服务不开机自启动
systemctl disable sshd
#当天关闭服务
systemctl stop sshd

#2合1
systemctl disable --now sshd
```

#### 3.1.3 重启

```sh
systemctl restart sshd
```

#### 3.1.4 检查

```sh
# systemctl status 服务名字

# 脚本使用的，检查服务状态的命令
systemctl is-active 服务名
systemctl is-enabled 服务
```

#### 3.1.5 故障

- 服务启动或重启失败了如何排查?

```sh
# 重启服务
systemctl restart sshd

#报错
Job for sshd.service failed because the control process exited with error code.
See "systemctl status sshd.service" and "journalctl -xe" for details.

1.通过journactl-xe或journactl-xe -u sshd找服务名字从下到上一点点看.
2.找关键词bad/error/failed关键文件配置文件
3.查看日志：
1)服务有日志看服务的日志mysq1日志在data目录下主机名.err结尾.
2)服务没有日志看/var/log/messages或syslog
```
