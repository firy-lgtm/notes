# day10 Linux专题-权限管理

## 一、权限管理

### 1.1 权限基础

- 权限：控制文件或目录访问，限制访问，rwx控制Linux权限

| 权限 | 说明                                                         | 代表数字 |
| ---- | ------------------------------------------------------------ | -------- |
| r    | 读取权限，查看文件内容权限或查看目录内容，需要配合x权限使用  | 4        |
| w    | 写入权限，是否可以修改文件内容或是否可以创建，删除，重命名文件权限，需配合x，r使用 | 2        |
| x    | 执行权限，运行命令，脚本或是否能进入目录，查看修改目录下文件属性 | 1        |
| -    | 没有权限                                                     | 0        |

### 1.2 一个文件，目录的权限是什么

```sh
-rw-r--r--  1 root root    590  3月 22  2022 zshrc

rw- 所有者的权限         
r--（中间）用户组的权限   
r--（最后）其他人的权限   
```

### 1.3 修改权限

- chmod change改变/修改mode(权限)修改权限rwx 655
- chown change改变/修改 owner(所有者)修改文件目录所有者用户组信息

使用数字修改权限

```sh
# 把文件lidao.sh的权限修改为777
chmod 777 lidao.sh（文件名） 

# 把文件lidao.sh的权限修改为000
chmod 000 lidao.sh（文件名）
```

```sh
# 修改文件的用户和用户组
chown   所有者.用户组    文件
```

注意：选项-R递归修改目录及目录下内容所有者.

使用字母修改权限

```sh
# 给lidao.sh文件的所有者添加执行的权限，用户组添加写入的权限，其他人添加执行的权限和写入的权限
chmod u+x,g+w,o+wx lidao.sh

# 给lidao.sh的所有者，用户组，其他人添加执行的权限
chmod a+x lidao.sh
chmod ugo+x lidao.sh

#记住这个案例即可给/etc/rc.d/rc.local文件(红帽)  ubt/debian /etc/rc.local (没有需要创建)
#增加执行权限
```

**注意：-R 递归修改权限（不可乱用）**

| 文件权限 | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| r        | read读取权限,查看文件内容权限.                               |
| w        | write写入权限/修改权限,是否可以修改文件内容.需要r权限配合.   |
| x        | execute执行权限(运行命令,运行脚本),需要r权限配合.(尤其是脚本)系统脚本,服务脚本要加上x权限. |
| -        | 没有权限                                                     |

### 1.5 目录rwx

| 文件权限 | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| r        | 查看目录内容,需要x权限配合.                                  |
| w        | 是否能在目录下创建,删除,重命名文件权限.需要x配合,但是有w,一般给rwx |
| x        | 是否能进入目录权限,是否查看或修改目录下文件属性权限.         |
| -        | 没有权限                                                     |

### 1.7 12位权限之特殊权限

- 这些特殊权限3个,经常遇到2个.
- 未来不要使用/创建这些特殊权限.

| 特殊权限     | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| suid setuid  | 例子:/bin/passwd命令,运行这个命令的时候相当于获取命令所有者(root)权限(所有人) |
| sticky粘滞位 | /tmp/拥有粘滞位的目录,777权限,每个用户只能管理自己创建的文件. |
| sgid setgid  | 运行这个命令的时候相当于获取命令用户组(root)权限(所有人)     |

```sh
chmod 4755 或u+s
chmod 1777 或o+t
```

### 1.8 特殊属性

- chattr,lsattrlinux系统(文件系统属性)
- a属性append只能追加,一直只能内容,不会涉及到修改删除.
- i属性immutalbe无敌的,不能修改.给命令进行配置.

```sh
chattr +i oldboy.txt # 添加文件/目录属性
chattr -i oldboy.txt # 删除文件/目录属性
```
